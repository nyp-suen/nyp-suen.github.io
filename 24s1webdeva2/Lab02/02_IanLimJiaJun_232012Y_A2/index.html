Introduction

Procedural animation is a type of computer animation.
It's used to automatically generate animation in real-time to allow for a more diverse series of actions than would otherwise be tedious using predefined animations.

This bug is created using 5 techniques. First Order Dynamics, Second Order Dynamics, Inverse Kinematics, Distance Constraints and Angle Constraints

First Order Dynamics

First order dynamics is a variation of interpolation that helps to give the object in motion the illusion of inertia.
It works by moving the object towards its goal by a factor of the distance between them and a damping constant every frame.

Second Order Dynamics!

Now this is where shit gets cool.
Second order dynamics is another variation of interpolation that helps to give the object in motion the illusion of inertia.
It works by adding on to first order dynamics by moving the object by a factor of the distance travelled in the previous frame and a bounce constant.
